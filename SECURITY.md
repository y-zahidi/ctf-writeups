# Security Policy

This repository is maintained by **Yassir Zahidi** — cybersecurity engineering student, builder of [`home-lab-siem`](https://github.com/y-zahidi/home-lab-siem).

## Reporting a vulnerability

If you find a security issue in this repository:

1. **Do not** open a public issue or pull request.
2. Email **yassirzahidi8@gmail.com** with subject line `security: ctf-writeups`.
3. Include the affected commit / tag, a minimal reproduction (PoC, payload, or steps), and your impact assessment.

I'll acknowledge within **48 hours** and aim to ship a fix or mitigation within **7 days** (high), **14 days** (medium), **30 days** (low). Coordinated disclosure timelines are welcome — say so in the first email.

## Scope

In scope: source code, configuration, CI workflows, container definitions in this repository.
Out of scope: third-party services this repo links to (report to the vendor), social-engineering against the maintainer's GitHub account (report to GitHub Security).

## Safe harbor

Good-faith research that follows this policy will not be pursued legally. Don't exfiltrate data, don't disrupt service, demonstrate impact with the smallest possible footprint.
